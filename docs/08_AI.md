# AI Layer

> **Status:** Design specification - optional feature; milestone 6.

## Overview

AI is an **optional enhancement** to NomaBot-not a hard dependency. Users may run fully offline with template messages, or connect cloud/local models for context-aware speech bubbles, smart reminders, and summarization.

**Critical rule:** AI runs only on the **desktop**. The ESP32 never calls models or stores API keys.

```text
Plugin / Service
       │
       ▼
  AIService  ──►  AIProvider (pluggable)
       │                │
       │                ├── OpenAI
       │                ├── Google Gemini
       │                ├── Anthropic Claude
       │                ├── Ollama (local)
       │                └── LM Studio (local)
       ▼
  Noma Runtime → device (text only)
```

## Design goals

| Goal | Approach |
|------|----------|
| Provider independence | Single `AIProvider` interface |
| Local-first option | Ollama / LM Studio as first-class |
| Privacy | Keys in OS credential store; user controls context |
| Graceful degradation | Personality **templates** when AI unavailable ([Offline Mode](./16_OFFLINE.md)) |
| Cost control | Token limits, caching, user opt-in per feature |

## AIService responsibilities

- Select active provider from settings
- Build prompts from **explicit context** (never silent full-disk scrape)
- Enforce rate limits and timeouts
- Emit `ai.response` events for plugins
- Never send secrets to device-only final display strings

### Supported providers (planned)

| Provider | Type | Typical use |
|----------|------|-------------|
| **OpenAI** | Cloud | GPT-4.x / GPT-4o mini |
| **Google Gemini** | Cloud | Gemini Pro / Flash |
| **Anthropic Claude** | Cloud | Claude 3.x |
| **Ollama** | Local | llama3, mistral, etc. |
| **LM Studio** | Local | OpenAI-compatible local server |

Configuration example (`config.json`):

```json
{
  "ai": {
    "enabled": true,
    "provider": "ollama",
    "model": "llama3",
    "base_url": "http://127.0.0.1:11434",
    "max_tokens": 120,
    "temperature": 0.7
  }
}
```

Cloud providers store API key references (`credential://openai`) resolved via Windows Credential Manager (or platform equivalent).

## Context model

AI features receive structured context-not raw screen capture by default.

```json
{
  "activity": {
    "app": "Code.exe",
    "title": "main.py - NomaBot",
    "idle_seconds": 12
  },
  "git": {
    "branch": "main",
    "dirty_count": 2
  },
  "reminders": [
    { "text": "Stand up", "due_in_minutes": 5 }
  ],
  "user_locale": "en-US"
}
```

Plugins and services assemble context. Users disable categories in Settings → Privacy.

### Privacy tiers

| Tier | Data sent to model |
|------|-------------------|
| **Off** | No AI calls |
| **Minimal** | App name + idle only |
| **Standard** | + plugin-approved metadata (branch, track title) |
| **Extended** | User-opt-in extras (future: window title filtering) |

## Use cases

### 1. Context-aware messages

Replace static strings with short bubble text:

```text
Input context: coding in VS Code, 45 min idle break overdue
Output: "Stretch break? You've been at it a while."
```

Render via `show_message`-character animation unchanged.

### 2. Smart reminders

`ReminderService` triggers event; AIService optionally rephrases:

```text
Reminder: "Doctor appointment 3pm"
AI (optional): "Heads up-doctor at 3."
```

### 3. Plugin-assisted summaries

GitHub plugin passes PR title list; AI produces one-line desk summary (user confirms before display).

### 4. Character personality

AI must respect pack **`personality.yaml`** traits (friendly vs sarcastic vs calm). Prefer loading `ai.system_addendum` from [Character System](./06_CHARACTER_SYSTEM.md#personality) over inline metadata duplicates.

```yaml
# nomabot/personality.yaml (excerpt)
ai:
  system_addendum: |
    You are NomaBot. Under 12 words. Encouraging, not noisy.
```

## Prompt structure

Standard chat format internally:

```text
System: {personality + safety guidelines + length limit}
User: {structured context JSON + task instruction}
```

Safety guidelines (built-in, not user-editable):

- No medical/legal/financial advice beyond gentle reminders
- No profanity in default mode
- Refuse to exfiltrate secrets or env vars

## Streaming

When `supports_streaming` is true, AIService may stream tokens to desktop UI preview before committing to device. Device receives **final string only** (ESP32 text layout is not streaming-friendly).

## Error handling

| Condition | Behavior |
|-----------|----------|
| Provider offline | Fallback template; tray notice |
| Timeout | Retry once; then fallback |
| Rate limit | Queue or drop with log |
| Invalid key | Settings prompt; disable AI for session |
| Content filter | Empty message + log |

## Caching

Optional hash cache keyed by `(provider, model, context_hash, task)` to reduce API cost. TTL configurable; user can clear cache in settings.

## Plugin integration

Plugins request AI via permission `ai.query`:

```python
result = await context.ai.complete_task(
    task="summarize_git_status",
    context={"branch": "main", "dirty": 3},
)
context.request_render(message=result.text)
```

AIService merges plugin context with global privacy rules.

## Local model recommendations

For offline users:

| Engine | Notes |
|--------|-------|
| Ollama | Easy install; default for local |
| LM Studio | GUI management; OpenAI-compatible API |

Document minimum RAM (8 GB+) for local models in user docs when milestone ships.

## Non-goals (v1 AI milestone)

- Voice input / output on device
- On-device LLM on ESP32
- Automatic screen OCR
- Sending conversation history to cloud without explicit opt-in
