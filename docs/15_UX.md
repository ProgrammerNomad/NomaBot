# User Experience

> **Status:** Design specification - engineering docs describe *how*; this doc describes *what the user feels*.

## Overview

NomaBot is a **calm desk presence**, not a notification firehose. Characters react to your work with warmth and restraint. The UX goal: you glance at the device and feel accompanied-not monitored, not nagged.

## Design pillars

| Pillar | Meaning |
|--------|---------|
| **Calm** | Most of the time, the character idle-animates quietly |
| **Contextual** | Reactions tie to what you are actually doing |
| **Brief** | Messages are short; bubbles disappear |
| **Optional depth** | Power users enable plugins, AI, multi-device |
| **Offline-safe** | USB-only mode feels complete, not broken ([Offline Mode](./16_OFFLINE.md)) |

## Emotional arc of a day

### Morning

```text
08:00  Scheduler fires
       │
       ▼
Robot wakes from sleep animation (slow stretch graph transition)
       │
       ▼
Speech bubble: "Good morning."  (3–5 s, then fade)
       │
       ▼
Returns to idle - subtle breathing loop
```

**Feel:** Gentle start. No calendar dump unless user enabled morning briefing plugin.

### Work session

```text
User opens VS Code
       │
       ▼
Crossfade idle → typing animation + laptop accessory
       │
       ▼
(Optional) Git plugin: small banner "main · 2 files" - Normal priority
       │
       ▼
User deep in flow → character stays in typing; no interruptions (Low/Background suppressed)
```

**Feel:** The device "notices" you are coding. It does not cheer every keystroke.

### Afternoon

```text
14:00  Scheduler: hydration reminder (Normal priority)
       │
       ▼
Bubble: "Water?" + optional coffee accessory nudge
```

**Feel:** Helpful nudge, not alarm.

### Meeting / focus

```text
Calendar plugin: meeting started (High priority)
       │
       ▼
Accessory: headphones or "busy" bubble style
       │
       ▼
Spotify + git events suppressed until meeting ends
```

**Feel:** Respect for focus time.

### Evening

```text
18:00  Optional walk reminder
       │
       ▼
22:30  Good night sequence
       │
       ▼
Sleep animation + dimmed backlight (if supported)
```

**Feel:** Closure to the day.

## Character presence levels

User setting: **Presence** (Settings → General)

| Level | Behavior |
|-------|----------|
| **Minimal** | Idle + critical notifications only |
| **Balanced** (default) | Activity + scheduler + Normal plugin events |
| **Expressive** | Full plugin reactions + AI messages |

## Message tone

Messages follow **character personality** ([Character System](./06_CHARACTER_SYSTEM.md#personality)):

| Character | Example line | Avoid |
|-----------|--------------|-------|
| NomaBot | "Nice progress." | Sarcasm, noise |
| Coding Cat | "…you're still here?" | Corporate cheer |
| Mini Monk | "Breathe." | Urgency, caps |

When AI is enabled, system prompts include personality block-AI cannot override "not noisy" traits.

## Visual UX rules

| Rule | Rationale |
|------|-----------|
| Max bubble duration 8 s default | Desk is not a ticker |
| No more than 1 bubble queue visible | Queue merges text |
| Critical uses banner style | Meeting, build failed |
| Animations blend ≥ 150 ms | No jarring pops |
| Sleep after 5 min idle (configurable) | Device feels alive but restful |

## Multi-device UX

| Device | Typical placement | Default feel |
|--------|-------------------|--------------|
| Office | Main monitor | Full Expressive |
| Living room | Shelf | Minimal + weather theme |
| Workbench | Serial-only | Coding Cat, no Spotify |

Device Manager uses friendly names-not `esp32-a1b2`.

## First-run experience

```text
1. Install desktop → tray icon
2. Plug in LILYGO T-Display S3 via USB
3. Auto-pair prompt → name device → default NomaBot character
4. "You're set." idle animation within 60 s
5. Optional: enable Git plugin (permissions explained)
```

No account required. No Wi-Fi required.

## Error UX (user-facing)

| Situation | User sees |
|-----------|-----------|
| Unplugged USB | Tray: "Office device offline" - not a modal |
| Plugin crash | Settings badge; character keeps idle |
| AI unavailable | Template message or silence-never error codes on device |
| Pack install fail | Character Manager: plain-language validator list |

## Accessibility (target)

- Readable bubble contrast per theme
- Optional reduce-motion (skip celebratory animations)
- Optional disable sounds
